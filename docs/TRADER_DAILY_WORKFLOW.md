## ⏱️ 定时任务与分时快照运维说明（V26.2 更新）

**前端与定时逻辑彻底分离**

- Streamlit 实盘界面（`ui/app.py`）已退化为**纯事件驱动**：按钮点击、页面展示、同步扫描；**不包含**后台定时器、异步轮询或分时快照调度。
- **严禁**指望「刷新网页」「多开标签」或「保持页面常开」来触发任何定时任务；这些行为**不会**替代守护进程。

**自动化职责归属（实盘机必须常驻）**

- **盘中定时行情快照（多时点 VR 等）**、**自动扫描链路**、**危险信号落库/展示依赖的数据写入**、**企微自动化推送** 等长驻逻辑，必须由**项目外的守护型进程**承担；与浏览器会话无关。
- 当前仓库中：
  - **`auto_sniper_daemon.py`**：实盘机 **唯一后台大脑**（见 `register_schedules()`）：含 **分时六槽快照**、`scan_async` 队列轮询、**P2/P3/P4/P5 扫描**、**晚间同步+P1 链**、**早盘补位**、**早安例行** 等。**勿**再并行启动已删除的独立 scheduler/sync_daemon。

**部署纪律**

- 使用项目根 **`start_server.bat` / `start_server.sh`**：先 **`auto_sniper_daemon.py`**，再 **`streamlit run ui/app.py`**，全机仅两进程，避免 DuckDB 多守护抢锁。
- 亦可将 **`auto_sniper_daemon.py`** 单独配置为 **开机自启**（systemd、pm2、任务计划程序等）。
- 修改 `config.yaml`、总控 `master_control.json`、企微 Webhook 后，**无需**依赖前端刷新即可被守护进程下一轮任务读取；但**实验室会话覆写**仍只存在于 Streamlit 进程内（见下文）。

## DuckDB 维护模式、一键维护与读连接（V26.2 上线前必读）

**`maintenance_mode`（全局维护锁）**

- 持久化字段位于 `data/runtime/state/master_control.json`（与企微总闸、守护巡航等同一文件），由 `core/master_control.py` 读写。
- 为 **`true`** 时：`auto_sniper_daemon.py` 主循环在 `schedule.run_pending()` 前避让，定时链中增量同步、P1 重建、P2–P5 扫描、分时快照、异步扫描队列、周维护等写库路径会主动跳过，避免与运维 **VACUUM / 离线压库** 争用文件锁。
- 维护结束后务必回到 **`false`**（运维脚本会自动解除）；若异常中断，可手工编辑该 JSON 或再次执行维护脚本。

**一键强制数据库维护**

- 指挥舱侧边栏 **「💾 数据底座」** → **「🚨 一键强制数据库维护」**：会预先打开维护锁并尝试释放当前 UI 进程的 DuckDB 连接，再在**独立进程**中运行 `tools/force_maintenance_vacuum.py`（Windows 下通常弹出控制台窗口）。
- 脚本流程概要：写维护锁 → 缓刑等待 → 安全终止含 `auto_sniper_daemon` 的 Python 进程 → `duckdb_vacuum_silent()`（短时独占写连接）→ 解除维护锁 → 按平台规则重新拉起守护进程。
- **建议**：投递后关闭**多余**的 Streamlit 标签页，维护完成后再重开页面；若控制台仍提示库被占用，先关掉本机所有 `streamlit` 页面再单独重跑脚本。

**短读连接与 UI**

- `data/db_core.py` 提供 `get_read_conn(read_only=True)` 等**短时上下文**连接；指挥舱部分查库与 `scan_engine` 中 **`signal_log` 历史聚合**等已改为「查完即关」，降低与守护进程长连接的锁竞争。
- UI 侧**禁止**用 `st.cache_resource` 长期缓存 DuckDB 连接；允许对 DataFrame / 元组等使用 `@st.cache_data(ttl=60)` 做结果缓存（详见 `ui/app.py` 头部说明）。

**更多索引**：全文件路径清单见 `docs/PROJECT_FULL_INVENTORY.md`；异步扫描部署见 `docs/DEPLOY_SCAN_ASYNC.md`。

## 开机三件事

1. **确认守护进程已运行**：本机已启动 `auto_sniper_daemon.py`，日志无持续报错。  
2. **打开 Streamlit** → 运行 `streamlit run ui/app.py`，进入 **航母指挥舱**（单页，无「策略实验室」Tab；实验室请用 `streamlit run run_lab.py` 另开进程）。  
3. **侧边栏**：选 **1档（全市场优选）** 或 **直通车（自选股 `p0_custom.txt`）**。  
4. 若昨日在 **策略实验室**（`run_lab.py`）把 1档改成了 **宽松档**，今日盘前请 **改回与生产一致的 Profile** 并视需要 **重新洗盘**，勿把实验参数当默认。

---

## 盘前（直通车用户）

1. **侧边栏** 上传或确认 **`p0_custom.txt`**（与 `constants.P0_FILE_PATH` 一致）。  
2. 回 **指挥舱** → **启动自选股洗盘**，待底仓表有票。

全市场 1档用户：在指挥舱 **启动全量洗盘** 即可。

---

## 早盘～盘中（2档 / 3档）

在 **指挥舱** 使用 **2档·竞价**、**3档·盘中** 按钮；关注 **综合分、战法、风险标签**。  
Regime 退潮时 **降总仓位**。  
（自动 P3 巡逻与推送由 **`auto_sniper_daemon.py`** 在交易窗内按调度执行，与界面手动点击互补，非替代关系。）

---

## 尾盘：指数熔断（14:30 后）

1. 在指挥舱 **漏斗区附近** 展开 **「大盘指数熔断（防空洞）」** Expander → 点 **刷新指数熔断状态**（具体文案以界面为准）。  
2. 若显示熔断且配置了 **enforce_block_p4**，**4档** 可能被拦截。  
3. 若需 **尾盘宽松 1档**：打开 **`run_lab.py`** 策略实验室，选 **P1 · relaxed**，调参后回指挥舱 **重新洗盘**。

---

## 4档

**指挥舱** → **4档·盘尾池扫描**（同步执行，等待转圈结束）。结合主池 / 观察池与风控提示自行决策。  
（原「背书池一秒核对」功能已移除，请用 **底仓表 + 综合分 + 团队纪律** 替代。）

---

## 15:05 与次日

- 若使用实验室 **宽松档**，**收盘前** 请 **恢复严格档/中性档** 并 **重新洗盘**，避免次日仍用实验阈值。  
- 系统 **不会** 自动清空 `session_state` 中的实验室覆写。

**指挥舱** → **5档·盘后真龙扫描**（时间名仅为按钮文案）。
